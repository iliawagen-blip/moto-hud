# Мото-ИЛС (moto-hud) — мастер-план развития

> Живой документ. Обновляется по мере развития проекта.
> Последняя ревизия: 2026-07-03.

---

## 1. Видение продукта

**Мото-ИЛС** — навигатор-«стеклянная кабина» для мотоцикла: крупный, читаемый через визор HUD
(манёвр, дистанция, скорость, прогноз-дорожка, курс), голосовые подсказки в гарнитуру шлема,
надёжная работа **без сети** и в фоне.

Ключевые отличия от обычного навигатора:
- считывается за долю секунды на скорости и через тонированный визор;
- голос манёвров приходит в гарнитуру шлема без «танцев с бубном»;
- работает офлайн (карты + маршрут), потому что связь в горах и на трассе часто отсутствует;
- учитывает специфику мото: извилистость дороги, камеры «в спину», крупная моторика в перчатках.

**Целевая платформа №1 — Android** (аудитория мото + раггед-телефоны). iOS — позже.

## 2. Текущее состояние (baseline, честно)

**Есть:**
- Единое веб-приложение `index.html` + эмулятор `sim.html`.
- HUD на CSS Grid: статус-строка, крупная параметрическая стрелка манёвра, улица, дистанция,
  крупная скорость, прогноз-дорожка (3D-проекция), опциональный компас N/E/S/W.
- Адаптивная раскладка (портрет / 3 колонки в альбоме), полноэкранный режим.
- Плавная отрисовка: интерполяция позиции между GPS-фиксами (dead reckoning) + `requestAnimationFrame`,
  батч-сборка SVG, оконный поиск ближайшей точки маршрута.
- Веб-API: `geolocation.watchPosition`, `speechSynthesis`, `wakeLock`, Fullscreen.
- Эмулятор движения с маршрутом из поворотов разной крутизны и разворотом.
- Деплой на GitHub Pages через GitHub Actions.

**Нет (разрывы до «настоящего мотонавигатора»):**
- ❌ Офлайн: **всё онлайн** — маршрут (OSRM), камеры/POI (Overpass), поиск (Nominatim).
- ❌ Устанавливаемость: нет `manifest.json` и service worker.
- ❌ Фоновый GPS: в браузере умирает через 30–60 с после сворачивания.
- ❌ Гарантированный голос в гарнитуру: нет audio focus / управления BT-профилем.
- ❌ Стабильный курс на малой скорости: heading только из GPS, без sensor fusion.

## 3. Стратегические принципы

1. **Два независимых трека.** Развиваем параллельно:
   - **(A) Платформа/оболочка** — как приложение доставляется и получает доступ к железу.
   - **(B) Карты + роутинг** — что делает его навигатором (и офлайн).
   Для мотонавигатора (B) важнее (A) и почти от него не зависит.
2. **Ничего не выбрасываем зря.** Порядок: PWA → Capacitor (та же веб-сборка) → точечный натив.
   Переписывание на Flutter/натив — только если продукт «взлетит».
3. **Соло-темп.** Каждая фаза даёт работающий результат и не блокирует следующую.
4. **Безопасность важнее фич.** Читаемость, голос, отсутствие отвлекающих действий — приоритет №1.
5. **Тестируемость.** Любая фича проверяется в `sim.html` до выезда.

## 4. Решения по стеку (с обоснованием)

| Слой | Решение | Почему |
|------|---------|--------|
| Оболочка | **Capacitor** (после PWA-обвязки) | 90% текущего кода сохраняется; графика лёгкая (SVG+rAF ~0.47 мс/кадр) — минус WebView не важен; публикация в стор; нативные плагины точечно |
| UI/рендер | Остаётся **HTML + SVG + rAF** | Уже плавно и дёшево; выгода Flutter CustomPainter для нас почти нулевая |
| Карты (офлайн) | **MapLibre GL Native** + тайлы **PMTiles** | Свободный, векторные тайлы, биндинги под все оболочки, обёртки под Capacitor |
| Роутинг (офлайн) | **Valhalla** или **GraphHopper** (граф региона) | Свободные, встраиваемые, есть мото-профили с извилистостью |
| Голос | `speechSynthesis` сейчас → **нативный TTS + audio focus** в Capacitor | Гарантия доставки подсказки в гарнитуру, приглушение музыки/звонка |
| Гарнитура | Нативный BT-плагин (HFP/A2DP, audio focus) | В вебе управление профилем невозможно — это мото-критично |
| Натив Kotlin | Отложен, точечно | Только под фон/сенсоры/Android Auto, когда появится потребность продукта |

## 5. Дорожная карта по фазам

### Фаза 0 — Стабилизация прототипа ✅ (в основном сделано)
**Цель:** надёжный, читаемый HUD и повторяемое тестирование.
- [x] Grid-HUD, параметрическая стрелка, прогноз-дорожка, компас.
- [x] Адаптив портрет/альбом, полноэкранный режим.
- [x] Плавность (rAF + интерполяция) и производительность отрисовки.
- [x] Эмулятор с разными поворотами и разворотом.
- [ ] Мелкая полировка: единый конфиг цветов/размеров, чистка мёртвого кода.
**Готово, когда:** HUD стабилен в симуляторе на всех типах манёвров, без пустых кадров.

---

### Фаза 1 — PWA-обвязка ✅ (сделано)
**Цель:** устанавливается с иконкой, интерфейс работает без сети.
- [x] `manifest.json`: имя, иконка (SVG `any` + `maskable`), `display: fullscreen`, ориентация, тема.
- [x] Service worker (`sw.js`): precache оболочки, версионируемый кеш, cache-first/stale-while-revalidate.
- [x] Мета-теги PWA, регистрация SW (пропуск в режиме эмулятора), offline-fallback интерфейса.
- [x] Кеш последней поездки (маршрут + камеры) в localStorage, переигрывание при офлайне к тому же финишу.
**Готово, когда:** приложение ставится на домашний экран Android, открывается офлайн (интерфейс),
последний маршрут доступен без сети. ✔ Проверено локально: SW `activated`, кеш оболочки наполнен,
оболочка отдаётся из кеша офлайн.
**Заметки:** icon в SVG (без бинарников); для «идеальных» иконок стора позже добавить PNG 192/512.
iOS-ограничения PWA — на этой фазе не целимся в iOS.

---

### Фаза 2 — Capacitor: настоящее Android-приложение (1–2 недели)
**Цель:** APK/AAB из текущего кода + фоновый GPS + wake.
- [ ] Инициализация Capacitor, сборка Android-проекта из веб-бандла.
- [ ] Плагин фоновой геолокации (проверить надёжность засыпания — **главный риск**).
- [ ] Foreground-service с уведомлением «навигация активна».
- [ ] Экран/wake-lock нативно, отключение засыпания.
- [ ] Разрешения (геолокация «всегда», батарея без оптимизации).
- [ ] Внутренний тестовый трек Play Console (closed testing).
**Готово, когда:** приложение из стора едет с заблокированным экраном/свёрнутым, GPS не отваливается.
**Риск:** качество плагина фонового GPS — прототипировать в первую очередь, до всего остального.

---

### Фаза 3 — Голос в гарнитуру + стабильный курс (1 неделя)
**Цель:** подсказки гарантированно звучат в шлем; курс не «плавает» на малой скорости.
- [ ] Нативный TTS через плагин + **audio focus** (ducking музыки, вклинивание при звонке).
- [ ] Проверка A2DP/HFP-маршрутизации на реальной гарнитуре шлема.
- [ ] Sensor fusion heading: магнитометр + гироскоп + GPS (нативный плагин/модуль).
- [ ] Приоритезация и антиспам подсказок (не повторять, не наслаивать).
**Готово, когда:** голос манёвра слышен в гарнитуре поверх музыки, курс стабилен на стоянке и в пробке.

---

### Фаза 4 — Офлайн-карты и офлайн-роутинг (2–4 недели) ⭐ ключевая ценность
**Цель:** навигатор работает без интернета.
- [ ] MapLibre GL с векторными тайлами (PMTiles); подложка под прогноз-дорожку/обзорную карту.
- [ ] Менеджер регионов: скачать/удалить область, показать размер и покрытие.
- [ ] Офлайн-роутер (Valhalla/GraphHopper), граф региона, мото-профиль (извилистость).
- [ ] Абстракция маршрутизации: онлайн-OSRM ↔ офлайн-роутер за единым интерфейсом.
- [ ] Пересчёт маршрута офлайн при сходе с трассы.
**Готово, когда:** маршрут строится и пересчитывается в авиарежиме на скачанном регионе.
**Заметка:** голосовой поиск адресов офлайн — сложно; допускаем онлайн-геокодер для поиска.

## 7. Риски и меры

| Риск | Влияние | Мера |
|------|---------|------|
| Плагин фонового GPS ненадёжен (засыпание) | Блокер навигации | Прототип в Фазе 2 **первым делом**, до остальных фич |
| Голос не идёт в гарнитуру / нет audio focus | Мотонавигатор бесполезен | Нативный TTS + тест на реальной BT-гарнитуре в Фазе 3 |
| Размер офлайн-тайлов/графа | UX/хранилище | Разбиение по регионам, PMTiles, показ размера до скачивания |
| Онлайн-сервисы (OSRM/Overpass) ограничивают public-эндпоинты | Нестабильность | Свой инстанс или офлайн-роутер (Фаза 4) |
| Разрастание одного `index.html` | Поддержка | Модуляризация при переходе на Capacitor-сборку |
| Соло-ресурс | Скорость | Жёсткая приоритезация: платформа → офлайн → фичи |

## 8. Метрики успеха

- **Читаемость:** ключевая информация считывается ≤ 0.5 с (субъективный тест на выезде).
- **Надёжность фона:** 0 «отвалов» GPS за 2-часовую поездку со свёрнутым экраном.
- **Голос:** 100% манёвров озвучены в гарнитуре без потери на музыке/звонке.
- **Офлайн:** маршрут строится и пересчитывается без сети на скачанном регионе.
- **Батарея:** приемлемый расход за час активной навигации (замер на устройстве).
