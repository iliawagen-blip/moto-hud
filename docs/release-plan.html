1. Стратегия: три канала, один код

КаналКомуКогдаФормат

Все три канала собираются из одного npm run build. Android — Capacitor; iPhone — только браузер (нативного iOS-приложения пока нет).

2. Этап 0 — общая подготовка (3–5 дней)

0.1 Release-ключ (один раз, до первого APK)

Создать keystore, хранить отдельно от репозитория (бэкап обязателен).

  • Один ключ → sideload APK → RuStore → все обновления
  • Потеря ключа = невозможность обновлять ru.vagin.motohud

0.2 Версионирование

В android/app/build.gradle:

  • versionName: 0.1.0-alpha.1, 0.1.0-alpha.2 …
  • versionCode: монотонно растёт (2, 3, 4 …)

В каждом тесте фиксировать: версия + git commit + дата.

0.3 Политика конфиденциальности

Нужна для RuStore; полезна и для web. Страница на GitHub Pages, например:

https://iliawagen-blip.github.io/moto-hud/docs/privacy.html

Минимум:

  • что собирается: GPS (на устройстве), избранное (localStorage), запросы к OSRM / Nominatim / Overpass;
  • что не собирается: аккаунты, серверная аналитика (если так);
  • зачем геолокация и уведомление «Навигация активна»;
  • контакт разработчика (email / Telegram).

3. Этап 1 — Web для друзей с iPhone (1–2 дня)

URL для рассылки

  • Демо без GPS: sim.html
  • Реальный GPS (ограниченно): index.html
  • HUD с автозапуском: index.html?sim=1
iPhone (Safari) — честные ограничения
ФункцияiPhone (web)Android (APK)
GPS в движении✅ экран включён✅ + фон
GPS с выключенным экраном❌✅ (FGS)
Wake Lock⚠️ нестабильно✅
Голос⚠️ web TTS✅ нативный TTS
«Как приложение»«На экран Домой»APK / RuStore

Рекомендация: sim для знакомства; для реальной поездки — экран не гасить, Safari на переднем плане.

Чек-лист web-теста (iPhone)

  1. Открыть sim → маршрут → HUD → повороты, голос (если не mute).
  2. Основной URL → разрешить геолокацию → маршрут рядом с домом.
  3. «Поделиться» → «На экран Домой» — полноэкранный запуск.
  4. Короткая поездка 10–15 мин с включённым экраном.
  5. Фидбек по шаблону: что сломалось / не слышно / курс плавает.

Доработки web до рассылки

  • Страница docs/privacy.html + ссылка в «Как пользоваться»
  • Короткая friends.html — Android: APK, iPhone: sim + ограничения
  • Проверить деплoy: push → gh-pages → ссылка живая

4. Этап 2 — APK для себя и друзей на Android (3–7 дней)

Сборка

npm run build
npx cap sync android

Android Studio → Build → Generate Signed App Bundle or APK → APK → release → подписать keystore.

APK: например releases/motohud-0.1.0-alpha.1.apk (Google Drive / Telegram, не обязательно в git).

Раздача друзьям

  1. Файл APK + инструкция: скачать → «неизвестные источники» → геолокация + уведомления → батарея без ограничений (Samsung/Xiaomi).
  2. Список тестеров: имя, модель, Android, версия APK.
  3. Канал фидбека: Telegram-чат или Google Form.

Smoke-тест (до раздачи)

2Маршрут → ПОЕХАЛИ → HUD☐
3Экран off 10 мин → GPS жив☐
4Свёрнуто 10 min → уведомление FGS☐

6TTS-баннер / установка голоса☐

Полевой тест (2–4 недели, ≥ 3 устройства)

Сценарии — docs/oem-gps-matrix.md:

  • A: экран погашен 30 мин
  • B: приложение свёрнуто 30 мин
  • C: поездка 2 ч суммарно
Критерий перехода к RuStore: GPS стабилен на ≥ 3 вендорах, нет крашей на старте HUD.

5. Этап 3 — RuStore (после альфа, ~1–2 недели)

5.1 Аккаунт разработчика

  • RuStore Консоль → VK ID
  • Физлицо — достаточно для бесплатного приложения без монетизации
  • ИП/юрлицо + УКЭП — для монетизации через RuStore

5.2 Загрузка сборки

RuStore принимает APK и AAB (см. инструкцию публикации):

APK друзьямRuStore
Подписьrelease keystoreта же
Packageru.vagin.motohudтот же
versionCodeбольше предыдущегострого больше

Для первой публикации проще APK (как у друзей). AAB — когда понадобится оптimизация размера.

5.3 Декларация разрешений

РазрешениеЗачем
ACCESS_FINE_LOCATIONнавигация, маршрут, камеры
FOREGROUND_SERVICE + LOCATIONGPS при выключенном экране
POST_NOTIFICATIONSстатус «Навигация активна»
WAKE_LOCKэкран не гаснет в HUD

5.4 Комментарий модератору (шаблон)

Авторизация не требуется.
1. Разрешить геолокацию и уведомления.
2. Дождаться GPS (чип «GPS» зелёный).
3. Финиш: 55.758, 37.618 или поиск «Кремль».
4. «Построить маршрут» → «ПОЕХАЛИ».
Демо без GPS: sim в браузере (ссылка в описании).
Политика: https://iliawagen-blip.github.io/moto-hud/docs/privacy.html

5.5 Риск модерации: «просто WebView»

RuStore не принимает приложения, которые только ведут на сайт. У Мото ИЛС есть нативное: FGS GPS, TTS, keep-awake, уведомления — это нужно явно описать в карточке. Основной UI не должен открывать внешний сайт.

5.6 Порядок публикации

VK ID→ Создать приложение→ Загрузить signed APK→ Обработка→ Карточка + privacy→ Декларация GPS→ Модерация→ Релиз

Обработанный файл хранится 14 дней — за это время отправить на модерацию или сохранить черновик.
