Направление 15 — Яндекс как источник маршрутов

Запрещено: парсинг HTML yandex.ru/maps, закрытые API Навигатора, steps из фонового Яндекс.Навигатора. Только waypoints из URL (rtext) + redirect short-link.

Схема 15.1 — Ссылка «Поделиться» → waypoints

Пользователь строит маршрут в Яндекс.Картах → копирует ссылку → Moto HUD парсит rtext → OSRM/Valhalla между точками.

UX: для совпадения с Яндексом — промежуточные точки каждые 3–5 км.

async function parseYandexRouteLink(rawUrl){
  let url = rawUrl.trim();
  if(SHORT_RE.test(url)){
    const res = await fetch(url, { redirect: 'follow', method: 'HEAD' });
    url = res.url;
  }
  const m = url.match(/[?&]rtext=([^&]+)/);
  if(!m) throw new Error('Нет rtext');
  return decodeURIComponent(m[1]).split('~').map(parseRtextPoint).filter(Boolean);
}

Направление 16 — HUD↔карта (двойной тап)

Цикл: HUD → MAP_OVERVIEW → MAP_ZOOM → HUD. Навигация (snap, голос, камеры) не останавливается.

РежимПоведение
HUDСтрелка + дорожка (как сейчас)
MAP_OVERVIEWfitBounds остаток маршрута, padding 40 px
MAP_ZOOMzoom 16–17, центр на GPS, обновление каждые 2 с

Компоновка (жёстко)

Блок скорости — ровно ⅓ экрана во всех режимах. Карта занимает оставшиеся ⅔ (верх в портрете, слева в альбоме).
