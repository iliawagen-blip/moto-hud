1. Видение продукта

Мото-ИЛС — навигатор-«стеклянная кабина» для мотоцикла: крупный HUD (манёвр, дистанция, скорость, прогноз-дорожка, курс), голос в гарнитуру шлема, работа без сети и в фоне.

  • Считывается за долю секунды на скорости и через тонированный визор
  • Голос манёвров в BT-гарнитуру без «танцев с бубном»
  • Офлайн-карты и маршрут — связь в горах часто отсутствует
  • Учитывает мото: извилистость, камеры «в спину», крупная моторика в перчатках
  • Кооперативность с Яндекс Навигатором, антирадаром, музыкой — не война за ресурсы

Целевая платформа №1 — Android. iOS — после вехи M6.

2. Текущее состояние

Уже есть

  • HUD: grid, стрелка, дорожка, компас, finish-info
  • RouteGeometry + профиль высот + curve speed warning
  • rAF, dead reckoning, GPX-replay в sim.html
  • PWA: manifest, service worker, кеш оболочки
  • Capacitor Android, фоновый GPS, keep-awake
  • Нативный TTS (MVP), TTS-health, упреждение по времени
  • Топливный ассистент ⛽ (OSM + ГдеБЕНЗ в native)
  • 6 тем оформления × день/ночь/авто
  • Модули esbuild → js/app.js, деплой GitHub Pages

Критичные разрывы

  • Офлайн: маршрут OSRM, POI Overpass, поиск Nominatim — всё онлайн
  • Фоновый GPS в браузере умирает через 30–60 с
  • Audio focus / BT-маршрутизация в гарнитуру не гарантированы
  • OEM-тесты фонового сервиса (≥ 3 вендора)
  • Closed testing Play Console
  • Курс: sensor fusion MVP, полный ROTATION_VECTOR — далее

3. Стратегические принципы

  1. Два трека: (A) платформа/оболочка и (B) карты+роутинг — параллельно; для мотонавигатора (B) важнее.
  2. Ничего не выбрасываем: PWA → Capacitor → точечный натив. Flutter — только если продукт «взлетит».
  3. Соло-темп: каждая фаза даёт работающий результат.
  4. Безопасность > фичи: в движении — только крупная моторика и «да/нет» голосом.
  5. Тестируемость: любая фича — сначала sim.html и реальные GPX-фикстуры.

5. Дорожная карта по фазам

Фаза 0 — Стабилизация прототипа

✓ в основном
  • Grid-HUD, стрелка, прогноз-дорожка, компас, адаптив, fullscreen
  • RouteGeometry ROADPATH этапы 1–4 (snap, Frenet, elev, curve warning)
  • GPX-replay, синтетический маршрут в sim, голос только на поворотах
  • Единые токены оформления, кэш для SVG
  • Регресс на полном наборе GPX-треков — расширять по мере выездов

Готово когда: HUD стабилен в sim на всех типах манёвров и на записанных треках.

Фаза 1 — PWA-обвязка

✓
  • manifest.json, SW precache, offline UI
  • Кеш последней поездки (localStorage MVP)
  • Миграция кеша → IndexedDB (перенесено в фазу 4)

Фаза 2 — Capacitor Android

◐ ~75% → фокус
  • Модуляризация, esbuild, www/, cap sync
  • Background geolocation, foreground service, wake-lock
  • Разрешения, онбординг «отключите оптимизацию батареи»
  • OEM-матрица ≥ 3 вендоров — oem-gps-matrix.md
  • Closed testing Play Console — чек-лист

Риск: Xiaomi/Huawei/Samsung убивают фон — тестировать в первую очередь.

Фаза 3 — Голос в гарнитуру + стабильный курс

◐ ~55%
  • Нативный TTS, очередь, антиспам, TTS-health / офлайн-голоса
  • DeviceOrientation + GPS (MVP), калибровка «восьмёрка», fallback
  • Упреждение подсказок по времени (секунды, не метры)
  • Кооперативный audio focus (ducking, очередь при занятом фокусе)
  • Проверка A2DP/HFP на реальной гарнитуре шлема
  • ROTATION_VECTOR нативно, приоритезация антиспама

Фаза 4 — Офлайн-карты и офлайн-роутинг

○ ключевая ценность → спайк
  • Спайк CLI: npm run spike:routers — router-spike.md
  • Выбор роутера: Valhalla vs GraphHopper vs BRouter
  • MapLibre + PMTiles, пайплайн Planetiler, менеджер регионов
  • Мото-профиль: веса surface/smoothness, профили «быстро/извилисто/красиво»
  • Абстракция OSRM ↔ офлайн; пересчёт при сходе ≤ 3 с
  • IndexedDB для кеша поездок и метаданных регионов

Пробки — только онлайн-планирование. В офлайн-навигации пробок нет и не обещаем.

Фаза 5 — Мото-функции

◐ ~15%
  • Топливный ассистент ⛽ (3 тапа: по маршруту / перестроить / отмена)
  • Камеры офлайн (OSM + ФВФ ГИБДД + открытые данные; geo-toggle для ЕС)
  • Запись треков (SQLite/JSONL 1 Гц → GPX через SAF)
  • POI по интересам (еда, кемпинг, сервис, виды) + голос «заправка/кофе/сервис»
  • Профили поездки и покрытия (асфальт / смешанный / бездорожье)
  • Round-trip «N часов / M км, поизвилистее»
  • Статистика поездки, опц. сенсоры, закат

Фаза 6 — Планировщик многодневных путешествий

○
  • MVP: маршрут + коридорный POI + разбивка по дням (лимит км)
  • Модель Trip → Day → Segment; базы, радиалки, бюджеты отклонения
  • Ручное редактирование, экспорт GPX / yandexnavi:// / текст
  • HUD: «день N, до ночёвки X км», следующий POI дня

Фаза 7 — UX-полировка и настройки

◐ ~40%

  • Переключатель тем в настройках + быстрая кнопка 🌓 на HUD (≥48 px)
  • Крупные тач-зоны, finish-info (ETA / время / дистанция)
  • Яркость/контраст под визор, единицы, ru/en
  • Деградация при потере GPS, «нет маршрута»
  • Локальные шрифты woff2 (сейчас @font-face-заглушки)

8. Сосуществование с другими приложениями

  • Audio focus: ducking музыки только на время фразы; очередь, если говорит Стрелка/Яндекс
  • HUD-оверлей (SYSTEM_ALERT_WINDOW): скорость + манёвр поверх чужого навигатора
  • Режим спидометр + камеры без активного маршрута
  • yandexnavi:// — легальный экспорт точки/маршрута во внешний навигатор
  • Запрет: парсинг данных Яндекса (карты, пробки, камеры) — только MapKit как альтернатива всему стеку
