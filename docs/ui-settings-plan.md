# План оптимизации UI (настройки и экран setup)

**Версия:** 2.0  
**Дата:** 2026-07-08  
**Контекст:** `#drawer-opts` — ~32 контрола в одном списке; setup — 7 секций `<details>` + дублирование избранного.  
**Ревизия:** по ревью v1.0 — телеметрия до перекройки, плоские секции вместо triple-nested details, hotfix НАСТР., пресеты, миграция.

## 1. Диагноз

### 1.1 Что перегружено

| Зона | Проблема |
|------|----------|

| **Подсказки в списке** | Длинные `hint` раздувают высоту |
| **Inline-стили** | Нет единого ритма отступов |

См. таблицу v1.0 (без изменений ID). Модули: `app-opts.js`, `hud-opts.js`, `elevation.js`, `curve-speed.js`, `fuel-config.js`, `telemetry-ui.js`.

### 1.3 Диагностика до перекройки (обязательно)

**Гипотеза распределения по секциям сейчас — на глазок.** Перед UI-2 перераспределить по данным.

| Задача | Реализация |

## 2. Информационная архитектура setup

### 2.1 Вертикаль экрана

```
[Шапка] GPS · Камеры
[Маршрут] Финиш · поиск · ⛽
[Мои места] до 5 + «Управление →»
[Карта] подложка · Построить
[Маршрут] карта · ПОЕХАЛИ (sticky)

─── ниже сгиба ───
[🗓 Планировщик]
[⚙ Настройки]     ← один <details>, внутри плоские секции
[❓ Помощь]
[💚 Поддержать]   PWA
```

Телеметрия — **внутри** «Разработчик», не отдельный drawer на верхнем уровне.

### 2.2 «⚙ Настройки» — двухуровневая схема (не triple-nested)

**Один** `<details id="drawer-opts">`. Внутри — **не** вложенные `<details>`, а секции с заголовками:

```html
<details id="drawer-opts" open>  <!-- open всегда при раскрытии «Настройки» -->
  <summary>⚙ Настройки</summary>
  <div class="drawer-body">
    <h3 class="opts-heading">Основное</h3>
    …контролы…
    <h3 class="opts-heading opts-heading--collapsed" data-section="hud">Экран в пути</h3>
    …
  </div>
</details>
```

**Почему не details в details в details:** потеря ориентации в перчатках; разное поведение `toggle` на Android Chrome / iOS Safari / Firefox. Тестировать на **реальных устройствах**, не только DevTools.

**Состояние секций:** «Основное» — **всегда видимо** при открытых настройках. Остальные секции — **сворачиваемые блоки** (кнопка-заголовок или один уровень `<details>` только для «Расширенные» и «Разработчик»). Состояние свёрнутости — `localStorage` `moto-hud-opts-sections`.

### 2.3 Разработчик

- Активация: **7 тапов** по версии в help/footer + запись `localStorage devMode=1`.
- `?dev=1` — только для **браузера**; в standalone PWA URL не сохраняется → не полагаться на query.
- В help: явная строка «Как включить режим разработчика».

## 3. Распределение контролов (черновик → уточнить по телеметрии)

### 3.1 Основное (всегда видимо)

| Контрол | Обоснование |
|---------|-------------|
| `opt-voice` | Каждая поездка |
| `opt-theme` | Визуальная идентичность |
| `opt-mode-day/night/auto` | Сегмент 3 кнопки (тип контрола **не менять** без миграции handlers) |
| `opt-cams` | Главный переключатель |
| `opt-path` | Прогноз-дорожка |
| **`opt-limit`** | Лимит скорости для камер — **каждый подкручивает под себя** |
| **Пресеты** | Город · Трасса · Тур (см. §3.2) |

### 3.2 Пресеты — таблица значений

Поведение: пресет **применяет пакет один раз**; дальше пользователь правит вручную. Переключение пресета **не** «привязывает» настройки к пресету.

**Confirm обязателен:** «Заменить текущие настройки пресетом „Трасса“?»

| Настройка | Город | Трасса | Тур |
|-----------|-------|--------|-----|
| `opt-voice` | вкл | вкл | вкл |
| `opt-cams` | вкл | вкл | вкл |
| `opt-back-only` | вкл | вкл | выкл |
| `opt-path` | выкл | вкл | вкл |
| `opt-crossings` | вкл | вкл | вкл |
| `opt-elev-profile` | выкл | выкл | вкл |
| `opt-curve-warn` | вкл | вкл | вкл |
| `opt-curve-strict` | strict | normal | relaxed |
| `opt-limit` | 60 | 90 | 90 |
| `opt-cam-speed-tol` | 10 | 15 | 20 |
| `opt-hud-status-mode` | tap | tap | tap |
| `opt-hud-finish-mode` | tap | always | tap |

Хранение последнего пресета: `localStorage` `moto-hud-preset-applied` (метка, не автосинхронизация).

### 3.3 Онбординг: `opt-back-only`

Часто выставляется **один раз** при первом запуске → вынести в **мастер после legal** (вместе с лимитом скорости): «Предупреждать только камеры в спину?» Да/Нет. В постоянных настройках остаётся в секции «Камеры».

### 3.4 Экран в пути

`opt-hud-status-mode`, `opt-hud-finish-mode`, блок ETA (3 чекбокса), `opt-crossings`, `opt-heading` + `btn-compass-cal`, `opt-elev-profile` (+ числа при вкл.).

### 3.5 Камеры и предупреждения

`opt-back-only` (если не задан в онбординге), `opt-curve-warn`, `opt-curve-strict`, `opt-cam-speed-tol`.  
**Расширенные:** `opt-tol`, `opt-nodir`.

### 3.6 Мотоциклы и топливо

`opt-fuel-count`, `opt-fuel-proxy` (+ tooltip «?»), позже профили мото из `trip-planner-plan.md`.

### 3.7 Тонкая настройка HUD

Шевроны, рельеф px/км/усиление.

### 3.8 Разработчик

`opt-telemetry`, `opt-track-record`, список сессий (бывший `#drawer-telemetry`).

## 4. HUD: quick sheet (после пресетов)

### 4.1 Поведение НАСТР.

| Условие | Действие |
|---------|----------|
| **P0 hotfix** | Любой тап → setup + открыть настройки + scroll |
| **UI-3** | На **остановке** (`dispSpeed` ≤ 15 км/ч): bottom sheet |
| **В движении** | Sheet **не открывать**; toast «Настройки — на остановке» |
| Долгий тап | **Не использовать** |

### 4.2 Содержимое sheet

- Голос, тема (горизонтальный выбор), камеры вкл/выкл
- **Пресеты** Город / Трасса / Тур (главная ценность sheet)
- «Все настройки →» → полный setup

**Не в sheet:** яркость экрана (нет Web API) — не обещать.

### 4.3 User-story UI-3

> На остановке водитель за 3 тапа применяет пресет «Трасса» без входа в длинный список.

## 5. Визуальная система

### 5.1 CSS

```css
.opts-heading { font-size: 13px; text-transform: uppercase; … }
.opt-row { min-height: 56px; … }  /* все строки — 56px, без 48px */
.opt-tooltip { … }  /* 2–3 строки + «Подробнее →» */
```

**WCAG / перчатки:** минимум **56×56 px** tap target для всех интерактивных элементов. Экономия 8 px не стоит промахов.

### 5.2 Stepper для чисел

- Реализация: `<button>` − / значение / `<button>` +, не CSS на `input[type=number]`.
- **Auto-repeat** при удержании +.
- Группа рельефа: grid 3 колонки.

### 5.3 Локализация (задел)

Summary и кнопки пресетов — с `gap` и `min-width`; немецкие подписи могут быть на 30–50% длиннее. Не фиксировать ширину кнопок пресетов жёстко.

## 16. Технические ограничения

1. **Не переименовывать `id="opt-*"`** (UI-0–UI-4).
2. **Не менять тип контрола** (checkbox↔radio, select↔custom) без обновления всех `addEventListener` в `setup.js` / `app-opts.js`.
3. Если останется один уровень `<details>` для «Расширенные» — тестировать `toggle` на nested handlers.
4. `S` + `save*ToStorage()` — единственный источник truth.
5. Sheet — thin wrapper над теми же setter'ами.
